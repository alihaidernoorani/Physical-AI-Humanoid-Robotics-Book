# MDX Compliance Checklist

This checklist ensures all MD and MDX files comply with constitutional requirements and are safe for Docusaurus build.

## MDX Syntax Compliance
- [ ] No JSX-breaking syntax (e.g., unexpected commas in attribute lists)
- [ ] Properly closed HTML tags
- [ ] Valid Markdown syntax
- [ ] Correct code block formatting (``` instead of ~~~)
- [ ] No unescaped special characters that break MDX parsing

## Content Compliance
- [ ] No reader-facing external references, citations, or bibliographies
- [ ] All content is self-contained without requiring external links
- [ ] Technical accuracy maintained throughout
- [ ] Pedagogical quality preserved

## Formatting Standards
- [ ] Consistent heading hierarchy (H1, H2, H3, etc.)
- [ ] Proper list formatting (consistent indentation)
- [ ] Code blocks with proper language specification
- [ ] Images with proper alt text and relative paths

## Docusaurus Compatibility
- [ ] All internal links are valid and use proper relative paths
- [ ] Sidebar navigation references are correct
- [ ] No broken cross-references between chapters
- [ ] Components import properly (Callout, Diagram, Exercise, Glossary, Quiz, Toggle)

## Module Structure Preservation
- [ ] Module names unchanged
- [ ] Chapter order preserved
- [ ] Module hierarchy maintained
- [ ] No scope creep or autonomous expansion
